# OpenNova app — screenshot generator (fastlane snapshot)

Generates the iOS app screenshots used by `../OpenNova-Manual.html`, driven by
the app's built-in **demo mode** so no server, mower or login is required.

- **Language:** English (`en-US`) · **Device:** iPhone 16 Pro Max
- Captures the 6 bottom tabs + key sub-screens (Mapping, Schedule editor,
  Mower settings, OTA).

## Important: this tooling depends on app source code

The generator lives here under `manuals/`, but it drives the real app at
`../../app`. A few **source edits in the app cannot move here** — they're part
of the app itself (and harmless: gated behind `EXPO_PUBLIC_SNAPSHOT`, off in
every normal/dev/EAS/production build):

| File in `app/` | Why it's needed |
|----------------|-----------------|
| `app/src/lib/snapshot.ts` | `SNAPSHOT_MODE` flag (`EXPO_PUBLIC_SNAPSHOT === '1'`) |
| `app/App.tsx` | skips login + boots demo mode in snapshot mode; tab `testID`s |
| `app/src/context/DemoContext.tsx` | `initialEnabled` prop |
| `app/src/screens/{MapScreen,ScheduleScreen,AppSettingsScreen}.tsx` | `testID`s on sub-screen entry buttons |

## How it works

1. The JS bundle is built with `EXPO_PUBLIC_SNAPSHOT=1` (passed via the
   Snapfile's `xcargs`). In that mode the app skips the login gate and boots
   into demo mode, so every screen is populated.
2. `fastlane/snapshot/SnapshotUITests.swift` taps through the app using stable
   `testID`s and calls `snapshot(...)`.
3. fastlane builds the scheme in **Release** (embedded bundle — Debug needs
   Metro) and captures each screen.

## One-time / after every `expo prebuild`

The app's `ios/` folder is gitignored and regenerated by `expo prebuild`, which
wipes the UI-test target. Re-create it with:

```bash
cd manuals/OpenNova
ruby fastlane/setup_snapshot.rb
```

If your Ruby lacks the `xcodeproj` gem, use fastlane's bundled copy:

```bash
cd manuals/OpenNova
FL=$(echo /opt/homebrew/Cellar/fastlane/*/libexec)
GEM_HOME="$FL" GEM_PATH="$FL" ruby fastlane/setup_snapshot.rb
```

(The `snapshots` lane runs this step automatically.)

## Run

```bash
cd manuals/OpenNova
fastlane snapshot          # uses fastlane/Snapfile
# or
fastlane ios snapshots     # lane: re-registers the target, then captures
```

Output lands in `manuals/OpenNova/fastlane/screenshots/` (gitignored). The
curated, colour-corrected (sRGB) screenshots actually used by the manual live in
`../img/`; the raw originals (incl. hand-supplied real-device shots) are in
`../source/`.

## Notes

- **Camera tab** shows the app's "no stream on stock firmware" empty-state.
- First run does a full Release build (slow once; cached afterwards).
- Real iPhone screenshots are HDR (Display P3 / PQ) and look washed out in
  print — convert to sRGB before embedding:
  `sips --matchTo "/System/Library/ColorSync/Profiles/sRGB Profile.icc" in.png --out out.png`
